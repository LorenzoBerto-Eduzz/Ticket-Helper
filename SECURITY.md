# Security Policy

## Supported Versions

Security fixes are applied on the active development line and included in the next release.

## Reporting a Vulnerability

If you find a security issue, do not open a public issue with exploit details.

Report privately to:
- Email: `lorenzo.berto@eduzz.com`

Please include:
- Steps to reproduce
- Impact
- Affected version/tag
- Suggested mitigation (if available)

## Release Integrity

- Official releases are published only from protected repository branches and tags.
- Installers should be downloaded from GitHub Releases, not from third-party mirrors.
